Delfrågor

Två frågor om CSS och koppling mellan HTML och CSS med id och class

  1. Vad menas med kaskad i CSS? Hur fungerar det?
  2. I HTML-taggarna kan man ha attribut med id och class och sedan referera till dem i CSS-koden. Vad är skillnaden, när ska man använda id och när ska man använda class?

Svar E1:

Flera CSS-egenskaper kan defineras flera gånger för samma HTML-element och ibland kan CSS-reglerna säga emot varandra. För att lösa detta dinns en rangordning som man kallar "kaskadordning", den säger i vilken ordning de olika reglerna ska gälla. När en webbläsare läser in ett HTML-dokument uppifrån och ned så är det den regeln som webbläsaren läst in sist som gäller.

Svar E2:

Id och class fungerar i princip på samma sätt. Det som skiljer dem åt är att du kan använda samma class till flera olika element på en webbsida och du kan inte direktlänka till en class.
Id brukar man använda på större områden till exempel om du har en sidomeny eller sidfot. Du tilldelar ett unikt id till någon HTML-tagg och därefter kan man formatera innehållet för det elementet hur man vill.
Class fungerar bättre att använda om du kanske vill markera vissa ord i ett textstycke. Till exmpel att vissa ord blir understrukna eller en annan färg.
Du kan även använda id till att direktlänka till ett visst område på en webbsida.
